# Agenda - Agentic Coding for Economists (Milan)

# questa è una prova

## Format
- **Capstone sprint** over 4 days
- **Demo-first, hands-on delivery** (minimal classical lecture)
- Participants build in parallel:
  1. their personal AI tooling stack/workflows
  2. their research project artifact

## Daily Flow (4h), approximative and subject to change
- 10 min: kickoff + day goals
- 20 min: live workflow demo
- 120 min: hands-on sprint block A (can be split in two if needed)
- 10 min: stand-up/checkpoint
- 60 min: hands-on sprint block B
- 20 min: demos, feedback, next-day setup

## Day 1 - Setup, GitHub Foundations, Capstone Start
- Demo: environment setup, Cursor-first workflow, modes (Agent/Ask/Custom), and agent mindset
- GitHub quick intro (practical): repo, clone/fork, commit/push, branch, PR, issues, labels
- Intro to coding with AI agents: Cursor modes (Agent/Ask/Plan/Debug), context control, rules/memories, AGENTS.md, .cursorignore, skills, subagents, cloud agents (formerly background agents), etc.
- Sprint goal: define project + research question + issue backlog + guardrails
- Required outputs:
  - project brief (1 page)
  - issue board (minimum 8 issues)
  - rules/memories + AGENTS.md + .cursorignore
  - first commit and push

## Day 2 - Research Pipeline Sprint
- Spec-driven development (SDD): research question -> research design -> research pipeline
- skills and agents for SDD
- Demo: idea -> literature map -> testable research path + context control
- Sprint focus: literature framing, hypothesis refinement, data plan, method choice, context discipline
- Required outputs:
  - one-page research design, with full specs (requirements, tasks, intent, etc.)
  - bibliography file (bibtex or equivalent) with literature review
  - prioritized implementation task list

## Day 3 - Parallel Agent Execution Sprint
- Demo: issue decomposition + subagents + skills lab + consistency/review pass
- Sprint focus: build prototype in parallel tracks (code + documentation)
- Required outputs:
  - full harness (skills, subagents, swarms)
  - working prototype (scripts/notebook/code/paper)
  - consistency/review log
  - merged PRs for core pipeline components


## Day 4 - Integration, Replication, Presentation
- Demo: from prototype to mini research package + replication clean-run
- Sprint focus: robustness, reproducibility, communication
- Completing the capstone project
- Required outputs:
  - mini replication package
  - replication README with clean-run command
  - 5-7 minute project presentation
  - next-30-day adoption plan
